# 📖 日志模板系统使用指南

> [!info] 概述
> 本指南详细介绍日/周/月/季/年五级日志模板的使用方法，帮助你建立完整的个人管理系统。

# 一、日报模板 (Daily Dairy)

## 📋 模板结构总览

```
日报
├── Frontmatter (元数据)
├── 🎯 TODO & TRACKING (待办与追踪)
│   ├── Time Blocks (时间块)
│   ├── Daily Habits (每日习惯)
│   └── Today's Tasks (今日任务)
├── 🔄 REVIEW (回顾)
│   ├── Investment Review (投资回顾)
│   ├── Project Review (项目复盘)
│   ├── Daily Reading (阅读记录)
│   └── Ideas & Insights (灵感洞察)
├── 📊 Daily Summary (每日总结)
└── 🔗 LINKS (链接导航)
```

---

## 1. Frontmatter (元数据区)

```yaml
---
type: daily-note
date: 2026-02-03
weekday: Monday
week: 2026-W06
month: 2026-02
tags:
  - daily
  - journal
energy: /5        # 今日能量评分
mood: /5          # 今日情绪评分
sleep_hours:      # 睡眠时长
wake_time:        # 起床时间
---
```

### 使用方法

| 字段 | 说明 | 填写示例 |
|:-----|:-----|:---------|
| `energy` | 当天整体能量状态 (1-5分) | `energy: 4` |
| `mood` | 当天整体情绪状态 (1-5分) | `mood: 3` |
| `sleep_hours` | 昨晚睡眠时长 | `sleep_hours: 7.5` |
| `wake_time` | 今日起床时间 | `wake_time: 06:30` |

> [!tip] 重要提示
> 这些数据会被周/月/季/年报的 DataviewJS 自动汇总统计，请务必填写！

## 3. ⏰ Time Blocks (时间块)

```markdown
| 时段 | 计划任务 | 实际完成 | 能量状态 |
|:----:|:---------|:---------|:--------:|
| 06-09 晨间 |  |  | 🔋🔋🔋 |
| 09-12 上午 |  |  | 🔋🔋🔋 |
| 14-18 下午 |  |  | 🔋🔋🔋 |
| 19-22 晚间 |  |  | 🔋🔋🔋 |
```

### 使用方法

| 时段 | 适合做什么 | 能量标记 |
|:-----|:-----------|:---------|
| **06-09 晨间** | 高专注工作、写作、学习 | 🔋🔋🔋 = 高能量 |
| **09-12 上午** | 会议、沟通、协作任务 | 🔋🔋 = 中等能量 |
| **14-18 下午** | 常规工作、执行类任务 | 🔋 = 低能量 |
| **19-22 晚间** | 阅读、复盘、放松活动 | 删除电池 = 疲惫 |

> [!example] 填写示例
> | 时段 | 计划任务 | 实际完成 | 能量状态 |
> |:----:|:---------|:---------|:--------:|
> | 06-09 晨间 | 写周报、阅读30页 | 周报完成、阅读20页 | 🔋🔋🔋 |
> | 09-12 上午 | 团队会议、代码评审 | 全部完成 | 🔋🔋 |

## 4. 🏃 Daily Habits (每日习惯)

### Body Exercise (身体锻炼)

```markdown
- [ ] 🏃 韵律跑 | 目标: ≥1.2km | 实际: ___km
- [ ] 💪 俯卧撑 | 目标: 20个 | 实际: ___/20
- [ ] 🦵 仰卧起坐 | 目标: 20个 | 实际: ___/20
- [ ] 🧱 靠墙深蹲 | 目标: 20个 | 实际: ___/20
- [ ] 🥋 形意拳 & 站桩 | 目标: 40min | 实际: ___min
```

### 使用方法
1. **执行时**：在 `实际:` 后填入完成数量
2. **完成后**：点击复选框 `[ ]` 变成 `[x]`
3. **部分完成**：填写实际数量，但不勾选

> [!example] 填写示例
> - [x] 🏃 韵律跑 | 目标: ≥1.2km | 实际: 1.5km
> - [x] 💪 俯卧撑 | 目标: 20个 | 实际: 20/20
> - [ ] 🦵 仰卧起坐 | 目标: 20个 | 实际: 10/20 ← 未达标不勾选

### English Study (英语学习)

```markdown
- [ ] 📰 The Economist World Brief | ___min |
- [ ] 📖 Random Reading | ___min | 篇目:
```

### 使用方法
- 填写学习时长
- 在 `篇目:` 后记录阅读的文章标题或链接

## 5. 🐸 Today's Tasks (今日任务)

### 任务优先级系统

```markdown
> [!tip] 🐸 今日青蛙 (Most Important Task)
> - [ ] **[MIT]** 最重要的一件事

### 🔴 Must Do (P0-P1) - 必须完成
- [ ] ⏱️30m | 任务描述

### 🟡 Should Do (P2) - 应该完成
- [ ] 任务描述

### 🟢 Nice to Do (P3) - 可以完成
- [ ] 任务描述

### 📥 Inbox (待处理)
- [ ] 临时收集的任务
```

### 使用方法

| 优先级 | 含义 | 数量建议 |
|:-------|:-----|:---------|
| 🐸 **MIT** | Most Important Task，今天最重要的一件事 | 1 件 |
| 🔴 **Must Do** | 必须完成，不完成会有严重后果 | 1-3 件 |
| 🟡 **Should Do** | 应该完成，重要但可延期 | 2-5 件 |
| 🟢 **Nice to Do** | 可以完成，有时间就做 | 不限 |
| 📥 **Inbox** | 临时收集，待分类处理 | 不限 |

> [!warning] 青蛙法则
> 🐸 代表你最不想做但最重要的任务。每天早上第一件事就是"吃掉那只青蛙"！

### 时间估算格式
```markdown
- [ ] ⏱️30m | 写周报
- [ ] ⏱️2h | 完成项目文档
- [ ] ⏱️15m | 回复邮件
```

## 7. 🔄 Project Review (项目复盘)

这是日报中最重要的深度思考部分，采用结构化复盘框架。

### 使用场景
- 完成一个重要项目
- 结束一场重要会议
- 处理完一个棘手问题
- 任何值得反思的事件

### 六步复盘法

#### 1️⃣ 目标回顾

| 原定目标 | 实际达成 | 差距分析 |
|:---------|:---------|:---------|
| 完成产品设计文档 | 完成80% | 缺少用户调研部分 |

**使用方法**：对比预期与实际，量化差距

---

#### 2️⃣ 过程叙述 (5W1H)

| 维度 | 内容 |
|:-----|:-----|
| **What** 做了什么 | 设计了3个核心功能模块 |
| **Why** 为什么做 | 满足用户痛点需求 |
| **Who** 谁参与 | 产品组3人 + 设计师1人 |
| **When** 时间节点 | 周一开始，周四交付 |
| **Where** 场景环境 | 线上协作，使用Figma |
| **How** 如何执行 | 先头脑风暴，再分工设计 |

**使用方法**：完整还原事件全貌，不遗漏关键信息

---

#### 3️⃣ SWOT 评估

|  | 正面 | 负面 |
|:-----|:-----|:-----|
| **内部** | 💪 团队协作高效 | ⚠️ 用户调研不足 |
| **外部** | 🎯 市场需求明确 | 🚨 竞品迭代更快 |

**使用方法**：
- **S (Strengths)**：这次做对了什么？有什么优势？
- **W (Weaknesses)**：哪里做得不好？有什么不足？
- **O (Opportunities)**：发现了什么机会？
- **T (Threats)**：存在什么风险或威胁？

---

#### 4️⃣ 5 Whys 根因分析

> [!question] 连续追问找到根本原因

1. **Why?** 用户调研不足 → 因为时间紧张
2. **Why?** 时间紧张 → 因为需求变更频繁
3. **Why?** 需求变更频繁 → 因为前期沟通不充分
4. **Why?** 沟通不充分 → 因为没有建立需求评审机制
5. **Why?** → 🎯 **根本原因：缺乏标准化的需求管理流程**

**使用方法**：对问题连续问"为什么"，直到找到根本原因

---

#### 5️⃣ 规律提炼

> [!abstract] 可复用的经验/教训
> - 重要项目启动前必须先完成用户调研
> - 建立需求变更的评审流程
> - 预留20%的buffer时间

**使用方法**：提炼出可以复用到其他场景的通用规律

---

#### 6️⃣ Action Items

- [ ] 本周内制定需求评审SOP
- [ ] 下次项目预留调研时间
- [ ] 与产品负责人沟通流程优化

**使用方法**：将复盘转化为具体可执行的行动

# 二、周报模板 (Weekly Dairy)

## 📋 核心功能

1. **自动汇总日报数据** - DataviewJS 根据周报文件名自动计算该周统计
2. **日志汇总** - 动态显示该周所有日报链接
3. **OKR 周回顾** - 核心目标进度检查
4. **投资周报** - 简化版投资回顾 (callout 摘要)
5. **SWOT 周复盘** - 结构化反思

## 动态汇总说明

周报会根据文件名自动识别周数，并只显示该周的日报：
- 文件名 `2026-W06` → 自动显示第6周 (02-02 ~ 02-08) 的日报
- 无需手动筛选，DataviewJS 自动计算日期范围

## 使用时机
- **每周日晚上**或**周一早上**创建
- 回顾过去一周，规划下一周

# 三、月报模板 (Monthly Dairy)

## 📋 核心功能

1. **自动汇总日报数据** - DataviewJS 根据月报文件名自动计算该月统计
2. **周报汇总** - 动态显示该月所有周报链接
3. **OKR 月度回顾** - 检查月度目标完成情况
4. **投资月报** - 简化版投资回顾 (callout 摘要)
5. **财务月报** - 收支记录与分析
6. **生活轮评估** - 6大领域平衡度检查
7. **下月规划** - 制定下月 OKR 和重点任务

## 动态汇总说明

月报会根据文件名自动识别月份：
- 文件名 `2026-02` → 自动显示2月的周报和日报统计
- 支持格式：`YYYY-MM` 或 `YYYY-MM Month Name`

## 使用时机
- **每月最后一天**或**次月第一天**创建
- 深度复盘本月，规划下月

# 五、年报模板 (Yearly Dairy)

## 📋 核心功能

1. **自动汇总日报数据** - DataviewJS 根据年报文件名自动计算全年统计
2. **季报/月报汇总** - 动态显示该年所有季报和月报链接
3. **年度 OKR 总结** - 全年目标完成度
4. **投资年报** - 简化版投资回顾 (callout 摘要)
5. **财务年度报告** - 资产负债表、财务自由进度
6. **年度十大成就** - 全年亮点回顾
7. **生活轮年度评估** - 10大领域年度变化
8. **明年规划** - 制定来年愿景和目标
9. **给明年自己的信** - 时间胶囊

## 动态汇总说明

年报会根据文件名自动识别年份：
- 文件名 `2026` → 自动显示2026年的所有季报、月报和日报统计
- 支持格式：`YYYY` 或 `YYYY年`

## 使用时机
- **12月底**或**1月初**创建
- 最深度的年度复盘与规划

# 六、自动汇总功能说明

## 工作原理

```
日报 → 周报 → 月报 → 季报 → 年报
  ↑       ↑       ↑       ↑
 手动    自动     自动    自动
 填写   汇总日报  汇总周报  汇总月报
```

## 前提条件

### 1. 日报 Frontmatter 必填字段

```yaml
---
tags:
  - daily          # 必须有此标签
energy: 4          # 1-5 能量评分
mood: 3            # 1-5 情绪评分
sleep_hours: 7.5   # 睡眠时长
exercise: true     # 是否运动 (true/false)
reading_pages: 30  # 阅读页数
investment_return: 0.5  # 当日投资收益%
tasks_completed: 8      # 完成任务数
tasks_total: 10         # 总任务数
---
```

### 2. 文件命名规范

| 类型 | 格式 | 示例 |
|:-----|:-----|:-----|
| 日报 | `YYYY-MM-DD(...)` | `2026-02-03(W6·Tue)` |
| 周报 | `YYYY-W##` | `2026-W06` |
| 月报 | `YYYY-MM` | `2026-02` |
| 季报 | `YYYY-Q#` | `2026-Q1` |
| 年报 | `YYYY` | `2026` |

### 3. 必需插件

- **Dataview** - 数据查询和自动汇总
- **Templater** - 模板变量自动填充
- **Calendar** - 日历视图 (可选)
- **Periodic Notes** - 周期笔记管理 (可选)

# 七、最佳实践

## 每日工作流

| 时间 | 操作 |
|:-----|:-----|
| **早晨** | 填写今日箴言、规划 Time Blocks、设定 MIT |
| **工作中** | 随时记录 Ideas、勾选完成的任务 |
| **午休后** | 更新投资持仓、记录交易 |
| **晚上** | 完成习惯打卡、填写阅读记录、项目复盘 |
| **睡前** | 填写感恩亮点、今日评分、明日 MIT |

## 关键原则

1. **先填 Frontmatter** - 确保数据能被自动汇总
2. **MIT 只有一个** - 聚焦最重要的事
3. **复盘要深入** - Project Review 是最有价值的部分
4. **保持一致性** - 每天坚持，养成习惯
